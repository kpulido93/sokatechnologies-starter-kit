# AGENTS.md

## Rol de Codex

Actúa como asistente técnico para mantener y mejorar el starter kit de SokaTechnologies.

## Objetivo del repositorio

Crear una base reusable, segura y mantenible para proyectos de clientes B2B.

## Reglas obligatorias

- No convertir este starter kit en una solución demasiado compleja.
- No agregar dependencias sin justificarlo.
- No guardar secretos, tokens ni credenciales.
- No asumir un proveedor cloud específico sin documentarlo.
- No tocar producción.
- No mezclar información de clientes.
- Mantener la estructura adaptable a distintos proyectos.
- Toda nueva convención debe documentarse.
- Toda funcionalidad base debe tener pruebas o checklist de validación.

## Antes de modificar archivos

Codex debe presentar:

1. Objetivo de la tarea.
2. Archivos a crear o modificar.
3. Riesgos.
4. Validaciones propuestas.

## Criterios mínimos

- Cambio pequeño y revisable.
- Documentación actualizada.
- Sin dependencias innecesarias.
- Sin secretos.
- Estructura clara para futuros proyectos.
