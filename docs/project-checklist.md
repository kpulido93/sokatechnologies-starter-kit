# Checklist de proyecto

## Objetivo

Usar esta lista para validar que un nuevo proyecto parte de una base ordenada, segura y mantenible.

## Descubrimiento inicial

- [ ] Objetivo del proyecto definido.
- [ ] Alcance inicial documentado.
- [ ] Stakeholders identificados.
- [ ] Restricciones de negocio y técnicas registradas.
- [ ] Riesgos iniciales anotados.

## Base técnica

- [ ] `README.md` adaptado al proyecto real.
- [ ] `docs/decisions.md` revisado y actualizado.
- [ ] Variables de entorno identificadas y documentadas.
- [ ] Estructura de carpetas validada para el caso de uso.
- [ ] Reglas de seguridad revisadas.

## Implementación

- [ ] El stack se eligió y quedó documentado.
- [ ] Las dependencias añadidas tienen justificación.
- [ ] Los módulos principales están definidos.
- [ ] Las integraciones externas están inventariadas.
- [ ] Los scripts operativos tienen propósito documentado.

## Calidad

- [ ] Existe estrategia mínima de pruebas.
- [ ] Se definieron criterios de aceptación.
- [ ] Se acordó qué validaciones son obligatorias antes de desplegar.
- [ ] Se documentaron límites y supuestos técnicos.

## Despliegue y operación

- [ ] La estrategia de despliegue está documentada.
- [ ] La gestión de secretos está resuelta fuera del repositorio.
- [ ] Existe procedimiento de rollback o contingencia.
- [ ] Se definieron responsables operativos.
- [ ] Se acordó observabilidad básica.

## Cierre de arranque

- [ ] No hay secretos ni datos de clientes en el repositorio.
- [ ] No se añadieron dependencias innecesarias.
- [ ] La documentación refleja el estado actual.
- [ ] El proyecto está listo para empezar implementación real.
